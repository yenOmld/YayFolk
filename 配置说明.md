# 配置文件说明

## 后端配置 (travelate-backend/src/main/resources/)

### application.yml
需要替换以下配置项：

#### 1. GitHub OAuth 配置
- `GITHUB_CLIENT_ID`: 您的 GitHub OAuth App Client ID
- `GITHUB_CLIENT_SECRET`: 您的 GitHub OAuth App Client Secret
- `GITHUB_REDIRECT_URI`: GitHub OAuth 回调地址，例如：`http://YOUR_SERVER_IP/api/oauth/github/callback`

#### 2. 前端地址
- `FRONTEND_BASE_URL`: 前端服务地址，例如：`http://YOUR_SERVER_IP`

#### 3. 邮箱配置（用于发送验证码）
- `username`: 您的 163 邮箱地址
- `password`: 您的 163 邮箱授权码（不是登录密码，需要在邮箱设置中获取 SMTP 授权码）

#### 4. 百度 AI 配置
- `app-id`: 百度 AI 开放平台 App ID
- `api-key`: 百度 AI API Key
- `secret-key`: 百度 AI Secret Key

#### 5. 百度翻译配置
- `app-id`: 百度翻译开放平台 App ID
- `secret-key`: 百度翻译 Secret Key
- `api-key`: 百度翻译 API Key

#### 6. 腾讯翻译配置
- `secret-id`: 腾讯云 Secret ID
- `secret-key`: 腾讯云 Secret Key

#### 7. 阿里云 OSS 配置（图片存储）
- `access-key-id`: 阿里云 AccessKey ID
- `access-key-secret`: 阿里云 AccessKey Secret
- `endpoint`: OSS Endpoint，根据实际区域修改
- `bucket-name`: OSS Bucket 名称
- `base-url`: OSS 访问基础 URL

### application.properties
需要替换以下配置项：

#### 数据库配置
- `spring.datasource.url`: MySQL 数据库连接地址
- `spring.datasource.username`: 数据库用户名
- `spring.datasource.password`: 数据库密码（**务必修改默认密码**）

## 前端配置

### 开发环境
前端通过 `vite.config.js` 中的代理配置连接后端，默认代理到 `http://localhost:8080`

### 生产环境
需要修改 `vite.config.js` 中的 `proxy` 配置，或者：
1. 将前端打包后的文件部署到 Nginx
2. 配置 Nginx 反向代理到后端服务

## 数据库初始化

1. 执行 `travelate_schema.sql` 创建数据库和表结构
2. 执行 `travelate_data.sql` 导入基础数据
3. 执行 `landmark_tabel.sql` 导入地标识别相关数据
4. 执行 `attractions_images.sql` 导入景点图片数据

## 其他服务依赖

### Redis
确保本地或服务器安装了 Redis，默认端口 6379

### 地标识别模型服务
- 默认地址：`http://localhost:5000`
- 需要单独部署地标识别模型服务

## 安全提示

1. **不要将敏感配置文件提交到版本控制系统**
2. 生产环境请使用强密码
3. 定期更换密钥和密码
4. 使用环境变量或配置中心管理敏感信息
