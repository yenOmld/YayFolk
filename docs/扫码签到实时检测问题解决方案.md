# 扫码签到实时二维码检测问题解决方案

## 问题描述

在商家扫码签到页面中，"捕获当前帧"功能可以正确识别二维码，但实时识别功能无法正常工作。即使点击"重新连接相机"，实时检测仍然不可用。

## 问题原因

### 根本原因
Chrome浏览器在某些版本中将 `BarcodeDetector` API 从默认启用改为了实验性功能，需要用户手动在浏览器设置中启用 `chrome://flags/#enable-experimental-web-platform-features`。

### 技术细节
- **原实现**：使用浏览器原生的 `BarcodeDetector` API 进行实时二维码检测
- **问题**：`BarcodeDetector` API 目前只在以下浏览器中默认支持：
  - Chrome 88+（但需要启用实验性功能）
  - Edge 88+（但需要启用实验性功能）
  - Opera 74+
  - Android Chrome 88+
- **局限性**：大多数用户不会手动启用实验性功能，导致实时检测功能失效

## 解决方案

### 采用 jsQR 库
使用纯JavaScript实现的 `jsQR` 库代替浏览器的 `BarcodeDetector` API。

### 实施步骤

#### 1. 安装依赖
```bash
npm install jsqr
```

#### 2. 修改导入
```javascript
import jsQR from 'jsQR'
```

#### 3. 更新变量定义
```javascript
// 移除
const detectorSupported = typeof window !== 'undefined' && 'BarcodeDetector' in window
let detector = null

// 添加
let scanCanvas = null
let scanContext = null
```

#### 4. 重写扫描函数
```javascript
async function scanFrame() {
  if (!videoRef.value || !cameraActive.value) {
    return
  }

  const videoWidth = videoRef.value.videoWidth || 0
  const videoHeight = videoRef.value.videoHeight || 0
  
  if (videoWidth === 0 || videoHeight === 0) {
    return
  }

  // 初始化canvas
  if (!scanCanvas) {
    scanCanvas = document.createElement('canvas')
    scanContext = scanCanvas.getContext('2d', { willReadFrequently: true })
  }

  // 确保canvas尺寸与视频匹配
  if (scanCanvas.width !== videoWidth || scanCanvas.height !== videoHeight) {
    scanCanvas.width = videoWidth
    scanCanvas.height = videoHeight
  }

  // 从视频帧绘制到canvas
  scanContext.drawImage(videoRef.value, 0, 0, videoWidth, videoHeight)
  
  // 获取图像数据
  const imageData = scanContext.getImageData(0, 0, videoWidth, videoHeight)
  
  // 使用jsQR解析二维码
  const code = jsQR(imageData.data, imageData.width, imageData.height, {
    inversionAttempts: 'dontInvert'
  })

  if (code && code.data) {
    const qrCode = normalizeCode(code.data)
    if (qrCode && qrCode !== lastDetectedCode) {
      lastDetectedCode = qrCode
      manualCode.value = qrCode
      await lookupBookingByCode(qrCode, { announceSuccess: true })
    }
  }
}
```

#### 5. 简化启动逻辑
```javascript
async function startCamera(manual = false) {
  // ... 相机初始化代码 ...

  // 直接启动扫描循环，不再检查浏览器API支持
  if (scanTimer) {
    clearInterval(scanTimer)
  }
  scanTimer = window.setInterval(scanFrame, SCAN_INTERVAL_MS)
  cameraMessage.value = ''
}
```

#### 6. 更新清理函数
```javascript
function stopDetectionLoop() {
  if (scanTimer) {
    window.clearInterval(scanTimer)
    scanTimer = null
  }
  lastDetectedCode = ''
  lastDetectedAt = 0
  scanCanvas = null
  scanContext = null
}
```

## 优势对比

### 原方案（BarcodeDetector）
- ✅ 原生API，性能较好
- ❌ 浏览器支持有限
- ❌ 需要用户手动启用实验性功能
- ❌ 兼容性差

### 新方案（jsQR）
- ✅ 纯JavaScript实现，无浏览器依赖
- ✅ 支持所有现代浏览器
- ✅ 无需用户配置
- ✅ 兼容性好
- ✅ 维护活跃，社区支持好

## 技术细节

### Canvas优化
- 使用 `willReadFrequently: true` 选项优化频繁读取操作
- Canvas尺寸只在必要时调整，避免频繁重分配
- 复用Canvas实例，减少内存分配

### 性能考虑
- 扫描间隔：700ms（`SCAN_INTERVAL_MS`）
- 防抖间隔：1600ms（`SCAN_DEBOUNCE_MS`）
- 避免重复检测相同的二维码

### 错误处理
- 视频未就绪时跳过扫描
- 图像数据获取失败时继续循环
- 检测失败不影响后续扫描
